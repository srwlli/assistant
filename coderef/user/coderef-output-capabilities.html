Code Intelligence Outputs

1. index.json
Location: .coderef/index.json

Purpose: Complete inventory of all code elements in the project

Contains: Functions, Classes, Components, Hooks, Types/Interfaces, Constants, Imports/Exports

Size: Typically 50KB - 2MB

{ "elements": [ { "type": "function", "name": "authenticateUser", "signature": "authenticateUser(username: string, password: string): Promise<User>", "file": "src/auth/index.ts", "line": 42, "dependencies": ["bcrypt", "jwt"], "exports": true } ] }
2. context.md
Location: .coderef/context.md

Purpose: Human-readable architecture overview

Contains: Project structure, architectural patterns, technology stack, core modules, entry points, configuration

Size: 20-100KB (500-2000 lines)

Use Cases: Quick onboarding, AI context for LLMs, architecture review

3. patterns.json
Location: .coderef/reports/patterns.json

Purpose: Detected code patterns and architectural decisions

Contains: Design patterns, API patterns, state management, error handling, auth patterns, database queries

Size: 10-50KB

4. coverage.json
Location: .coderef/reports/coverage.json

Purpose: Test coverage analysis

Contains: Tested/untested functions, test file locations, coverage by module

Size: 5-20KB

5. complexity/
Location: .coderef/reports/complexity/

Purpose: Code complexity metrics

Contains: Cyclomatic complexity, cognitive complexity, nested code analysis, hotspots

Files: complexity-summary.json, high-complexity.json, complexity-by-module.json

Size: 10-50KB total

Diagram Outputs

6. dependencies.mmd
Location: .coderef/diagrams/dependencies.mmd

Purpose: Dependency graph in Mermaid format

Rendering: GitHub, VS Code, Mermaid Live Editor

graph TD A[AuthService] --> B[UserRepository] A --> C[JWTService] B --> D[Database]
7. dependencies.dot
Location: .coderef/diagrams/dependencies.dot

Purpose: Dependency graph in GraphViz DOT format

Rendering: GraphViz tools, online DOT viewers

Export Formats

8. graph.json
Location: .coderef/exports/graph.json

Purpose: Complete graph export (nodes + edges)

Size: 100KB-5MB

Use Cases: Custom graph analysis, Neo4j integration, network visualization

9. jsonld/
Location: .coderef/exports/jsonld/

Purpose: Semantic web export (JSON-LD format)

Use Cases: Knowledge graphs, semantic search, RDF/SPARQL integration

Foundation Documentation

10. README.md
Location: .coderef/generated-docs/README.md

Purpose: Project overview and quick start guide

Sections: Description, Tech Stack, Getting Started, Structure, Features, Configuration, Common Tasks

Size: 10-30KB (200-500 lines)

11. ARCHITECTURE.md
Location: .coderef/generated-docs/ARCHITECTURE.md

Purpose: Detailed architectural documentation

Sections: System Overview, Core Components, Data Flow, Module Structure, Design Patterns, ADRs, Integration Points, Security

Size: 30-100KB (500-1500 lines)

12. API.md
Location: .coderef/generated-docs/API.md

Purpose: API reference documentation

Sections: Overview, Authentication, Endpoints, Request/Response, Error Codes, Rate Limiting, Versioning, Examples

Size: 20-80KB (300-1000 lines)

13. COMPONENTS.md
Location: .coderef/generated-docs/COMPONENTS.md

Purpose: Component inventory (UI projects only)

Sections: Component Hierarchy, Shared Components, Pages, Layouts, Props Reference, State Management, Patterns

Size: 30-100KB (400-1200 lines)

Generated for: React, Vue, Angular, Svelte projects

Standards Documentation

14. COMPONENT-INDEX.md
Location: coderef/standards/COMPONENT-INDEX.md

Purpose: Searchable component inventory with props and usage examples

15. UI-STANDARDS.md
Location: coderef/standards/UI-STANDARDS.md

Purpose: UI pattern standards discovered from codebase

Contains: Button styles, form patterns, modals, layouts, colors, typography, spacing

16. BEHAVIOR-STANDARDS.md
Location: coderef/standards/BEHAVIOR-STANDARDS.md

Purpose: Behavioral pattern standards

Contains: Error handling, loading states, form validation, navigation, state persistence

17. UX-PATTERNS.md
Location: coderef/standards/UX-PATTERNS.md

Purpose: UX flow and interaction standards

Contains: User flows, permission handling, RBAC patterns, multi-step forms, data tables, search/filter

Workorder Tracking Files

18. context.json
Location: coderef/workorder/{feature-name}/context.json

Purpose: Feature requirements and constraints

19. communication.json
Location: coderef/workorder/{feature-name}/communication.json

Purpose: Workflow status tracking

20. plan.json
Location: coderef/workorder/{feature-name}/plan.json

Purpose: Implementation plan with task IDs and phases

21. DELIVERABLES.md
Location: coderef/workorder/{feature-name}/DELIVERABLES.md

Purpose: Completion metrics (LOC, files, commits, time, contributors)
