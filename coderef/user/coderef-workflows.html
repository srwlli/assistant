CodeRef Workflows Guide

14 core workflows for feature implementation, documentation, and quality assurance

1. Complete Feature Implementation (End-to-End)

1-8+ hours High Complexity
Phase 1: Plan (5-10 min)
  • /create-workorder → Gather context (interactive Q&A)
  • Analyze project (coderef-context intelligence)
  • Create 10-section plan.json
  • Validate (score >= 90 recommended)
Phase 2: Execute (1-8 hours)
  • /execute-plan → Generate TodoWrite tasks
  • Activate expert persona (/ava, /marcus, etc)
  • Implement with code context
  • Update task status as work completes
Phase 3: Document (2-5 min)
  • /update-deliverables → Capture git metrics (LOC, commits, time)
  • /record-changes → Auto-detect git changes, update CHANGELOG.json
  • /update-docs → Bump version, update README, CLAUDE.md
Phase 4: Archive (1 min)
  • /archive-feature → Move to coderef/archived/, update archive index
Output: Fully implemented, tested, documented, and archived feature

2. Quick Idea Capture (Stub Workflow)

30 seconds Low Complexity
  • /stub → Captures idea title + description
  • Optionally captures conversation context
  • Saves to C:\Users\willh\Desktop\assistant\coderef\working\

Later: /create-workorder → Loads stub.json as seed data, continues with full planning workflow

Output: stub-{feature-name}.json in centralized backlog

3. Multi-Agent Feature Implementation

Variable Very High Complexity
Setup
  • /create-workorder → Create plan with parallel phases, set multi_agent: true
  • /generate-agent-communication → Creates communication.json, defines agent scopes and forbidden files
Execution (parallel)
  • Lloyd (Coordinator): /assign-agent-task agent_number=1
  • Agent 1 (Ava): Works on frontend tasks
  • Lloyd: /assign-agent-task agent_number=2
  • Agent 2 (Marcus): Works on backend tasks
  • Lloyd: /assign-agent-task agent_number=3
  • Agent 3 (Quinn): Works on tests
Verification
  • Lloyd: /verify-agent-completion agent_number=1
  • Lloyd: /verify-agent-completion agent_number=2
  • Lloyd: /verify-agent-completion agent_number=3
Aggregation
  • /aggregate-agent-deliverables → Combines metrics from all agents
  • /archive-feature → Complete
Output: Feature implemented by coordinated team of specialized agents

4. Planning Only (No Implementation)

5-10 min Medium Complexity
Option A: Full workflow
  • /create-workorder → Interactive context gathering, project analysis, plan creation + validation
Option B: Manual workflow
  • /gather-context → Create context.json manually
  • /analyze-for-planning → Generate analysis.json from coderef
  • /create-plan → Combine context + analysis → plan.json
  • /validate-plan → Quality check (0-100 score)
  • /generate-plan-review → Human-readable review report
Output: coderef/workorder/{feature}/plan.json (validated, ready for execution)

5. Documentation Generation

2-5 min Low-Medium Complexity
  • Foundation docs (README, ARCHITECTURE, API, SCHEMA, COMPONENTS): /generate-docs → Generates all 5 foundation docs
  • Single doc: /generate-individual-doc template_name="readme"
  • Quickref guide (interactive): /generate-quickref
  • Standards documentation: /establish-standards → Scans codebase for UI/UX/behavior patterns, creates 4 standards markdown files
Output: Professional documentation in coderef/foundation-docs/

6. Standards & Compliance

5-15 min (setup) + 10 sec (check) Medium Complexity
One-time setup
  • /establish-standards → Scans codebase for patterns, creates coderef/standards/*.md files
Full codebase audit
  • /audit-codebase → Scans all files, generates compliance report, includes fix suggestions
Pre-commit check (fast)
  • /check-consistency → Only scans modified files (git auto-detect), returns pass/fail for CI/CD, severity threshold: critical/major/minor
Output: Standards docs + compliance reports + CI/CD integration

7. Changelog Management

1-2 min Low Complexity
Manual entry
  • /add-changelog → Provide version, type, description, files, etc. Adds structured entry to CHANGELOG.json
Agentic (auto-detect from git)
  • /record-changes → Auto-detects changed files (git diff), suggests change_type from commits, calculates severity from scope, shows preview for confirmation, creates entry
Output: Updated coderef/CHANGELOG.json with structured entries

8. Risk Assessment

30 seconds Low Complexity
  • /assess-risk → Describe proposed change
  • AI evaluates 5 dimensions: Breaking changes, Security, Performance, Maintainability, Reversibility
  • Returns 0-100 risk score
  • Go/no-go recommendation
  • Optional: Compare multiple options
Output: Risk assessment report with recommendations

9. Test Automation

Variable Medium Complexity
  • Run tests with coverage: /run-tests → Runs pytest, generates coverage report, returns pass/fail status
  • Discover available tests: /discover-tests → Lists all test files and functions
  • Check test health: /test-health → Analyzes test quality metrics
Output: Test results + coverage reports

10. Workorder Tracking & Audit

10-30 seconds Low Complexity
  • Log new workorder: /log-workorder → Adds entry to global workorder-log.txt (Format: WO-ID | Project | Description | Timestamp)
  • Query workorder log: /get-workorder-log → Filter by project, workorder pattern, date. Returns entries in reverse chronological order
  • Audit all plans: /audit-plans → Scans coderef/workorder/, checks plan format, progress, staleness. Returns health score (0-100)
  • Track agent status: /track-agent-status → Shows real-time agent coordination dashboard, feature-level or project-wide view
Output: Audit trails + progress dashboards

11. Persona Activation

Instant Low Complexity
  • Activate built-in persona: /ava (Frontend specialist), /marcus (Backend specialist), /quinn (Test specialist), /taylor (General purpose), /lloyd (Orchestrator)
  • Generic activation: /use-persona name="ava"
  • Check current persona: /get-active-persona
  • Deactivate: /clear-persona
  • Create custom persona: /create-persona → Guided workflow, auto-generates system prompt
Output: Activated expert persona with specialized knowledge

12. CodeRef Intelligence Workflows

5-60 seconds Low Complexity
12a. Initial Project Scan (One-Time Setup)
  • Complete structure (all 16 outputs): python scripts/populate-coderef.py /path/to/project
  • Quick foundation only (2 files): ./scripts/scan-all.py /path/to/project

Output: .coderef/ directory with 16 output types

12b. Dependency Analysis
  • coderef_query query_type="calls" target="AuthService"
  • coderef_query query_type="imports-me" target="ThemeProvider"
  • coderef_impact element="UserModel" operation="refactor"

Output: Dependency graphs + impact analysis

12c. Pattern Discovery
  • coderef_patterns pattern_type="handlers"
  • coderef_patterns pattern_type="decorators"
  • coderef_coverage format="detailed"

Output: Code patterns + test coverage analysis

12d. Visual Diagrams
  • coderef_diagram diagram_type="dependencies" format="mermaid"
  • coderef_diagram diagram_type="calls" depth=3 format="dot"

Output: Mermaid or Graphviz diagrams

13. Agent Handoff

30 seconds Low Complexity
  • /generate-handoff-context → Reads plan.json, analysis.json, git history
  • Auto-populates 80%+ context fields
  • Creates claude.md for next agent
  • Modes: full (comprehensive) or minimal (quick)
Output: coderef/workorder/{feature}/claude.md with agent context

14. Feature Inventory

10 seconds Low Complexity
  • /generate-features-inventory → Scans coderef/workorder/ + coderef/archived/
  • Returns structured list with status, progress, workorder tracking
  • Formats: JSON or markdown
  • Optional: save to file
Output: Complete feature inventory (active + archived)

Workflow Combinations (Common Patterns)

Pattern 1: Quick Feature (No Plan)

Implement code → /update-deliverables → /record-changes → /archive-feature

Pattern 2: Plan-First Feature

/create-workorder → /execute-plan → /update-deliverables → /record-changes → /archive-feature

Pattern 3: Research & Plan

/stub → [Research] → /create-workorder → [Review plan] → /execute-plan

Pattern 4: Documentation Refresh

/establish-standards → /audit-codebase → /generate-docs → /record-changes

Pattern 5: Pre-Commit Quality Gate

/check-consistency → [Fix violations if any] → git commit

Workflow Decision Tree

Do you have a clear plan? ├─ No → /stub (capture idea) → /create-workorder (plan it) └─ Yes → Do you need multi-agent coordination? ├─ Yes → /create-plan multi_agent=true → /generate-agent-communication → [Assign tasks] └─ No → /execute-plan → [Implement] → /update-deliverables Is the feature complete? ├─ Yes → /update-deliverables → /record-changes → /update-docs → /archive-feature └─ No → Continue implementing with persona support (/ava, /marcus, etc) Do you need documentation? ├─ Foundation docs → /generate-docs ├─ Single doc → /generate-individual-doc ├─ Quickref → /generate-quickref └─ Standards → /establish-standards Do you need to check quality? ├─ Risk assessment → /assess-risk ├─ Standards compliance → /check-consistency ├─ Plan quality → /validate-plan └─ Test coverage → /run-tests

Total: 14 primary workflows + 5 combination patterns
