Handoff: Tracking API Implementation for Dashboard

To: Dashboard Agent (coderef-dashboard team)
From: Orchestrator
Status: Ready for Implementation
Priority: CRITICAL (blocks all dashboard features)
Date: 2025-12-26

Your Mission

Implement the file system-based API layer that powers the dashboard's workorder and stub tracking system.

Why This Matters:

  • Dashboard routes (/coderef-assistant, /coderef-sources) need live data
  • You're building the data foundation everything else depends on
  • This unblocks parallel work: you build API, UI team builds UI

What You're Building

3 API Endpoints

GET /api/stubs
  ↓ Returns: All stubs from assistant/coderef/working/

GET /api/workorders
  ↓ Returns: All workorders from assistant/coderef/workorder/

GET /api/projects
  ↓ Returns: All registered projects from projects.config.json

File System Scan Pattern

assistant/coderef/
├── working/              ← Scan for stub.json files
│   ├── coderef-explorer/
│   │   └── stub.json
│   └── persona-system/
│       └── stub.json
│
└── workorder/            ← Scan for plan.json + DELIVERABLES.md
    ├── auth-system/
    │   ├── plan.json
    │   ├── context.json
    │   └── DELIVERABLES.md
    └── dashboard-api/
        └── plan.json

Key Insight: Folder existence = workorder exists. Files inside are optional metadata. This "folder-based discovery" pattern is critical to understand.

Response Format Example

{
  "stubs": [
    {
      "feature_name": "coderef-explorer-endpoints",
      "description": "Wire coderef folders into dashboard",
      "priority": "high",
      "status": "stub",
      "folder_path": "assistant/coderef/working/coderef-explorer-endpoints"
    }
  ],
  "workorders": [
    {
      "workorder_id": "WO-AUTH-SYSTEM-001",
      "feature_name": "user-authentication",
      "status": "implementing",
      "progress": 67,
      "folder_path": "assistant/coderef/workorder/auth-system"
    }
  ]
}

Success Criteria

  • ✅ All 3 endpoints return valid JSON
  • ✅ Folder-based discovery works (scans directories)
  • ✅ Graceful handling of missing files (stub.json, plan.json optional)
  • ✅ Fast response time (< 200ms for typical project)
  • ✅ Can handle 50+ workorders/stubs without performance issues

Next Step: Once these endpoints are live, the UI team can build the dashboard views that consume this data.
